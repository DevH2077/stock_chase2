# 🔄 자동 버전 업데이트 가이드

푸시할 때마다 버전 배지가 **한국 시간(KST)**으로 자동 업데이트됩니다.

## 📝 버전 형식

- 형식: `v` + 년(2자리) + 월(2자리) + 일(2자리) + 시(2자리) + 분(2자리)
- 예시: `v2512301638` (2025년 12월 30일 16시 38분, 한국 시간)

## 🚀 사용 방법

### 방법 1: npm 스크립트 사용 (권장)

```bash
cd version_1
npm run push
```

이 명령어는 다음을 자동으로 실행합니다:
1. 버전을 한국 시간으로 업데이트
2. 변경된 파일을 git add
3. 커밋 생성
4. GitHub에 푸시

### 방법 2: 수동 실행

```bash
cd version_1
npm run prepush
git push
```

### 방법 3: Git Hook 사용 (선택사항)

Windows에서 Git Bash를 사용하는 경우:

```bash
cd version_1
powershell -ExecutionPolicy Bypass -File setup-git-hook.ps1
```

이후 `git push`만 실행해도 자동으로 버전이 업데이트됩니다.

## ✅ 확인 사항

- ✅ 버전은 **한국 시간(KST)**으로 표시됨
- ✅ 푸시 시점의 시간으로 고정됨
- ✅ 새로고침해도 버전이 변경되지 않음
- ✅ 다음 푸시 시에만 버전이 업데이트됨

## 📌 중요

- 버전은 **빌드/배포 시점**에 설정됨
- 실시간으로 갱신되지 않음
- 사용자가 새로고침해도 동일한 버전 유지
- 한국 시간(UTC+9) 기준으로 표시됨

## 🔧 문제 해결

### 버전이 업데이트되지 않아요

1. `node update-version.js`를 직접 실행해보세요
2. `index.html`과 `app.js` 파일이 수정되었는지 확인하세요
3. Git에 변경사항이 추가되었는지 확인하세요 (`git status`)

### 한국 시간이 아닌 다른 시간이 표시돼요

- `update-version.js`가 올바르게 한국 시간을 사용하도록 설정되어 있습니다
- 서버 시간대가 다를 경우, 로컬에서 실행하면 한국 시간으로 표시됩니다
